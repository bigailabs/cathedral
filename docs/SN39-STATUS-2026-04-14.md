# SN39 Subnet Status Report -- 2026-04-14

## Overview

Polaris is running its own validator and miner on Bittensor Subnet 39 (formerly Basilica), backed by a self-hosted incentive API that replaces the original Covenant-operated `api.basilica.ai`. The subnet code is forked and maintained at [github.com/bigailabs/substrate](https://github.com/bigailabs/substrate).

## What's Running

### Incentive API (api.polaris.computer)

All 7 endpoints the Cathedral validator needs for weight-setting are live and serving real data:

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/v1/incentive/config` | GPU categories, window hours, pricing params |
| GET | `/v1/prices/baseline` | Baseline GPU $/hr per category (A100, H100, H200, B200, B300) |
| GET | `/v1/prices/tokens` | TAO + SN39 alpha token prices from chain |
| GET | `/v1/incentive/cus` | Compute unit ledger rows in epoch window |
| POST | `/v1/incentive/cus` | Validator submits new CU rows |
| GET | `/v1/incentive/rus` | Revenue unit ledger rows in epoch window |
| POST | `/v1/incentive/slash` | Slash a miner node's CUs/RUs |

Auth: `X-Validator-Hotkey` header whitelist. Token prices fetched from bittensor chain with 120s cache. CU/RU ledger is in-memory with async locks (will move to DB when volume warrants it).

## Blocker: Validator Stake

**The validator cannot set weights on-chain.** The chain returns `NotEnoughStakeToSetWeights`.

- **Required:** 1,000 TAO staked (global `StakeThreshold` parameter)
- **Current stake:** 0 TAO
- **Available across all wallets:** ~1.68 TAO

Once 1,000 TAO is staked to the iota1 hotkey on SN39, the full pipeline activates:

```bash
btcli stake add --wallet.name iota1 --hotkey default --netuid 39 --amount 1000 --subtensor.network finney
```

## What Happens After Staking

1. Validator `set_weights` succeeds (currently retrying every ~30s)
2. Metagraph sync picks up miner UID 115 into validator's local DB
3. Miner connects via gRPC `RegisterBid`
4. Validator generates CU records, posts to our incentive API
5. Weight allocation shifts from burn-only to rewarding our miner
6. Miner earns emissions proportional to compute provided

## For Other Validators

To point your Cathedral validator at the Polaris incentive API, add to your `validator.toml`:

```toml
api_endpoint = "https://api.polaris.computer"
cu_generator_enabled = true
```

The API authenticates via `X-Validator-Hotkey` header. Contact us to get your hotkey whitelisted.
